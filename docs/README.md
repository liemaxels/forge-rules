# docs/

This folder stores output documents generated by Forge Agents during the development process.

---

## Document Types

| Document | Generated by | Naming Convention |
|----------|-------------|-------------------|
| Product Brief | CEO Agent | `PRODUCT_BRIEF_[ProjectName]_[YYYY-MM-DD].md` |
| Architecture | Architect Agent | `ARCHITECTURE.md` (project root) |
| Review Report | Reviewer Agent | `REVIEW_PR[N]_[YYYY-MM-DD].md` |
| Test Report | Tester Agent | `TEST_REPORT_[Feature]_[YYYY-MM-DD].md` |
| Security Audit | Security Agent | `SECURITY_AUDIT_[Feature]_[YYYY-MM-DD].md` |
| A11y Audit | A11y Agent | `A11Y_AUDIT_[Feature]_[YYYY-MM-DD].md` |
| Performance Audit | Performance Agent | `PERF_AUDIT_[Feature]_[YYYY-MM-DD].md` |
| UX Review | UX Agent | `UX_REVIEW_[Feature]_[YYYY-MM-DD].md` |

---

## Usage

When an agent produces a document, save it here with the naming convention above.

These documents serve as:
- **Audit trail** — what decisions were made and why
- **Onboarding material** — new team members can read the history
- **Review reference** — reviewers can check against the original brief
- **Retrospective input** — what was planned vs what was built

---

## What Goes in Project Root vs docs/

| File | Location |
|------|----------|
| `ARCHITECTURE.md` | Project root (living document, always current) |
| `CONVENTIONS.md` | Project root (living document) |
| `README.md` | Project root |
| `CHANGELOG.md` | Project root |
| Product Briefs | `docs/` (historical record) |
| Review Reports | `docs/` (historical record) |
| Audit Reports | `docs/` (historical record) |

---

## .gitignore Note

These documents should be committed to your project repository.
They are documentation, not secrets.

Exception: If a document contains sensitive business information (revenue targets, user research with PII), store it outside the repository.
